MCP Server

Agents that speak MCP can call Cotrex natively instead of shelling out. Cotrex runs as an MCP server over stdio and exposes a run tool that returns structured execution events.

cotrex mcp        # JSON-RPC 2.0 over stdio

Register with Claude Code​

claude mcp add cotrex -- /absolute/path/to/cotrex mcp

The run tool​

Input:

{ "command": "cargo test", "llm": false }
  • command (string, required) — the command line.
  • llm (boolean, optional) — compress output into an insight for this call.

Result content is the machine channel verbatim — the same as the CLI: rtk output lines followed by a single result footer (and an insight line when a failure was analyzed):

Compiling cotrex v0.1.0
{"type":"result","status":"ok","code":0}

isError is set when the command exits non-zero.

The set_agent tool​

cotrex installs a graphify code-map skill for the agent in use. When it can't detect which agent that is (no terminal to prompt, nothing in config or the environment), a run result includes a note asking the model to identify itself. The model then calls set_agent:

{ "agent": "codex" }

cotrex persists the platform and installs the graphify skill for it in the background. This is the no-TTY equivalent of the interactive agent prompt — the model tells cotrex what it's running in.

Protocol notes​

Cotrex implements a focused subset of MCP: initialize, tools/list, tools/call, and ping, over newline-delimited JSON-RPC 2.0 on stdio. In MCP mode stdout is the protocol channel — execution output is captured internally and only returned inside the tool result.
